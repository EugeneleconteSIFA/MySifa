Process de développement multi-dev — proposition

Isolation par slot pour que la panne d'un dev ne bloque plus les autres.
MySifa · 16 juillet 2026
Auteur : Eugène + Claude
Réf. : CLAUDE.md § « Stratégie v1/v2 »
Aujourd'hui

Un seul env de test partagé

  • Toutes les branches feature convergent vers staging — c'est ce qui alimente v1.
  • Cron VPS /etc/cron.d/mysifa-v1-pull pull chaque minute, restart mysifa-v1.
  • DB isolée v1 (production-v1.db), resync nightly depuis prod à 02:00 UTC.
Point de douleur — si un dev merge un truc qui pète v1 (502, migration cassée, import fail), tous les autres devs sont bloqués pour tester leur feature, et la promotion vers prod est gelée jusqu'à ce que le fautif corrige.
Demain

Un slot dev par personne + v1 d'intégration

  • 3 slots fixes (alpha/beta/omega) préprovisionnés, attribués à un dev par un fichier /etc/mysifa/dev-slots.conf.
  • Chaque slot track une branche dédiée (dev-alpha, dev-beta, dev-omega). Push force autorisé — c'est jetable.
  • Chaque slot a sa DB (production-alpha.db…), resync nightly depuis prod comme v1 déjà.
  • v1 devient l'env d'intégration — le dev n'y pousse que quand sa feature est propre sur son slot.
Bénéfice — une erreur 502 sur v-alpha ne touche que Dev A. v-beta, v-omega, v1 et la prod continuent de tourner. La promotion prod ne dépend plus de qui a cassé quoi côté sandbox.

Mise en place — 4 étapes, ~1 journée VPS

01
Provisionner 3 slots
Clone du repo × 3, .env par slot (ENV_NAME, DB_PATH, PORT), 3 services systemd + 3 vhosts nginx.
02
Branches tracking
Créer dev-alpha, dev-beta, dev-omega depuis staging. Push force autorisé (branches jetables, non protégées).
03
Cron pull + resync
Copier mysifa-v1-pull.sh et mysifa-v1-resync-db.sh en trois variantes, une par slot, échelonnées.
04
Bandeau + attribution
Étendre le bandeau html.py (dev-alpha jaune, etc.), /etc/mysifa/dev-slots.conf pour mapper slot → email dev.
Points ouverts — nommage des vhosts (alpha.mysifa.com vs alpha.dev.mysifa.com) · SSL wildcard vs certif par sous-domaine · reset button "remettre mon slot sur staging" dans /settings.
